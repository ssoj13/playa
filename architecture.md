У нас есть несколько больших частей приложения:

# Internals

1. IO: системные функции загрузки и записи media
  - Load
    - LoadExr
    - LoadVideo
  - Write:
    - WriteExr
    - WriteVideo

2. EventBus:
  - Шина по которой приложение передаёт информацию внутри себя, от одних частей к другим.

3. Node, Comp: основной building block нашего видеопайплайна.

# Panels:

* Project: содержит Comps
  - struct Project, которая содержит всю логику приложения
  - ProjectWindow UI - окно проекта
  - Data: HashMap or {Key:Comp} for fast lookup: storage for all project comps/subcomps, everything.
  - Cache - глобальный кэш для всего приложения
  - Workers - Global Thread Pool for generic tasks with lambdas
  - Composer: structure that owns the GPU buffers and composes layers.
  - Prefs - Attrs с атрибутами преложения и всех окон, которое сериализуется в JSON и десереализуется из JSON обратно на старте.
  - 

* Timeline - отображает Comps как слои и содержит элементы управления слоями
  - Окно разделено на две части по вертикали: левук с элементами управления и правую, отображающую набор клипов на таймлайне:
  - Справа:
    - Сама полосу таймлайна сверху окна со шкалой кадров и курсором timeslider - вертикальная линия
    - пространство под таймлайном - для children слоёв, которые содержит этот Comp.
  - Слева:
    - сверху: Start, Play/Pause, End, Loop checkbox.
    - внизу: элементы управления слоями, находящиеся ровно напротив самих слоёв справа.
  - Правая часть мэппит пространство таймлайна в кадры и обратно, и все отрисовки происходят во "Frame space".
  - Левая кнопка мыши изменяет текущее время при левом клике на таймлайн сверху
  - Выделяет слои и перемещает их, триммит их если кликнуть на край по рабочему пространству кадров (меняет атрибуты слоя - начальный кадр по сути)


* Viewport - принимает кадр и показывает его в окне внутри виджета. Окно можно панорамировать и изменять его размер мышью как в Nuke.
  - Принимает Frame / &Frame
  - Использует OpenGL
